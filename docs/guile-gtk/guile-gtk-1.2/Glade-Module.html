Glade Module

The Glade interface module can be used with 

     (use-modules (gtk-1.2 glade))

A call to glade_init is made automatically, an application doesn't need to do that itself.

The following rules apply to the various Glade types.

GladeXML
GladeXML is a sub-class of GtkObject so the notes concerning it apply (see Gtk Module). The following field accessors are provided,

glade-xml-filename gx Function
glade-xml-txtdomain gx Function


The following standard Glade functions have particular notes for the Guile Gtk interface.

glade-xml-get-widget gladexml [name] Function
The default name is #f, meaning the top level widget in gladexml.

glade-xml-new filename [root] Function
The default root is #f, meaning create the whole interface defined in filename.

glade-xml-signal-connect-full gladexml name [proc] Function
glade-xml-signal-autoconnect-full gladexml [proc] Function
As per the Gtk signal handlers, there's no "user data" parameter to proc. proc is called (proc handlername object signalname signaldata connectobject after?).

If proc is not given, or is #f, the default is to connect signalname on connectobject (or object if no connectobject) to the procedure handlername. handlername is evaluated as Scheme code, and so can simply be the name of a Scheme procedure, or it can be a lambda form or similar evaluating to a procedure.


The following standard Glade functions are not provided.

glade-init Function
There is no separate init function to call, all initializations are done automatically when the (gtk-1.2 glade) module is loaded.
